# Button Component

The Button component is a fundamental interactive element used throughout the design system. It provides consistent styling and behavior across different variants and sizes.

<CalloutBox type="info">
The Button component is built with accessibility in mind and supports keyboard navigation, screen readers, and proper focus management.
</CalloutBox>

## Basic Usage

<ComponentDemo title="Basic Button Examples">
  <Button>Default Button</Button>
  <Button variant="secondary">Secondary Button</Button>
  <Button variant="outline">Outline Button</Button>
</ComponentDemo>

```tsx
import { Button } from '@boombox/components';

export function MyComponent() {
  return (
    <div>
      <Button>Default Button</Button>
      <Button variant="secondary">Secondary Button</Button>
      <Button variant="outline">Outline Button</Button>
    </div>
```

## Accessibility

The Button component follows WAI-ARIA guidelines:

- **Keyboard Navigation**: Supports Tab navigation and activation with Enter/Space
- **Screen Reader Support**: Proper labeling and state announcement
- **Focus Management**: Clear focus indicators and proper focus handling
- **Loading States**: Announces loading state to screen readers

<CalloutBox type="success">
**Accessibility Best Practices:**
- Always provide meaningful button text or aria-label
- Use the appropriate button type (button, submit, reset)
- Ensure sufficient color contrast for all variants
- Test with keyboard navigation and screen readers
</CalloutBox>

## Examples

### Form Submission

```tsx
import { Button } from '@boombox/components';
import { useState } from 'react';

export function FormExample() {
  const [isSubmitting, setIsSubmitting] = useState(false);

  const handleSubmit = async () => {
    setIsSubmitting(true);
    try {
      // Simulate API call
      await new Promise(resolve => setTimeout(resolve, 2000));
      console.log('Form submitted successfully');
    } finally {
      setIsSubmitting(false);
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      {/* Form fields */}
      <Button 
        type="submit" 
        loading={isSubmitting}
        loadingText="Submitting..."
      >
        Submit Form
      </Button>
    </form>
  );
}
```

### Confirmation Dialog

```tsx
import { Button } from '@boombox/components';

export function ConfirmationDialog({ onConfirm, onCancel }) {
  return (
    <div className="flex gap-3">
      <Button variant="outline" onClick={onCancel}>
        Cancel
      </Button>
      <Button variant="primary" onClick={onConfirm}>
        Confirm
      </Button>
    </div>
  );
}
```

## Design Guidelines

<CalloutBox type="info">
**When to Use:**
- Primary actions (save, submit, continue)
- Secondary actions (cancel, back, edit)
- Destructive actions (delete, remove)
- Navigation actions (next, previous)
</CalloutBox>

<CalloutBox type="warning">
**Avoid:**
- Using too many primary buttons in one view
- Using buttons for non-actionable content (use links instead)
- Unclear or ambiguous button labels
- Buttons that don't provide clear feedback
</CalloutBox>
